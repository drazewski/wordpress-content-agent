# Taxonomy rules

- Respect the existing taxonomy rules gathered during `wp-onboarding` when the target project already has categories or tags.
- Assign one main category whenever possible.
- Choose 2-5 tags that reflect the article's concrete topic.
- Prefer existing tags over inventing new ones.
- Create a new tag only when it names an important concept, place, product, organization, or framework not already covered.
- Avoid tags that are too broad to be useful, like `news`, `article`, or `topic`.

# How to decide

1. Read the article title, excerpt, headings, and main argument.
2. Identify the article type: guide, analysis, research summary, opinion, case study, review, announcement, or other.
3. Match the article to the closest category.
4. Select tags that describe:
   - the core subject,
   - the method or format,
   - the domain,
   - any essential named entities.
