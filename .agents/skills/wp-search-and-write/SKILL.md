# Finding information

- The user may provide a topic, keywords, or a short brief.
- Search the web for specific, credible, high-signal sources.
- Prefer primary sources, reputable publications, institutional websites, technical documentation, and well-sourced specialist articles.
- Treat clickbait, vague summaries, AI-slop, and shallow rewrites as low-quality material.

# Building context and the article base

- Build a working set of 3-5 useful sources before drafting.
- Compare sources against each other instead of trusting the first match.
- If source quality is unclear, ask the user before building the article on top of it.
- Respect any instructions stored in the topic folder inside `source/`.
- Summarize findings as a factual base first; shape the final article only after the research is coherent.

# Rules

- Use this skill only when `project-context.md` allows external research or when the user explicitly asks for it.
- Cite sources clearly when the final article requires source attribution.
- Do not use external research to override stronger local source material without saying so.
